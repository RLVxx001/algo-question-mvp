# 前端说明

前端是无依赖静态工作台，由同一个 Python 服务托管。

## 入口

```text
http://127.0.0.1:18081/
```

## 当前页面能力

- 通过全局“新建题目”入口打开生成弹窗，选择知识点、难度、数量和是否使用模型。
- 查看当前运行状态，包括 LLM 是否已配置、模型名、接口地址、生成数量上限和验证轮数上限。
- 选择题面语言，默认中文，可切英文。
- 选择是否使用分步流程，并配置哪些步骤需要人工确认。
- 生成题目。
- 在独立题目库中按关键词、来源和题面语言筛选，并显示当前结果数量。
- 在流程页签查看每一步状态，并确认继续。
- 在编辑页签修改题面、格式、约束、样例、标签、题解、标准解、暴力解和数据生成器。
- 在编辑页签增删样例行，并在保存前维护每组样例的输入/输出。
- 保存编辑后清空旧审查、旧验证和旧导出状态，要求重新审查/验证/导出。
- 调整验证轮数和单进程超时时间。

## 多轮生成行为

开启分步流程时，系统不会一次性生成完整题目包。它会先生成当前步骤内容；如果该步骤需要人工确认，就停下来。用户保存修改并继续后，后续约束、样例、解法和数据生成器才会基于最新内容生成。
- 查看题目列表和题目详情。
- 查看题面、题解、标准解、暴力解、数据生成器。
- 触发静态审查。
- 触发样例校验和随机对拍。
- 以结构化方式查看相似题风险、审查摘要、验证摘要、失败用例、导出目录和导出阻断原因。
- 刷新页面后重新读取已持久化的审查、验证和导出阻断状态。
- 复制失败用例输入，并对单个失败输入调用复跑接口。
- 仅在审查和验证都通过后导出题目包；失败时保留报告并提示导出被阻止。
- 在导出报告中下载题目包 ZIP。
- 删除当前题目，并同步清理流程记录和导出文件。
- 查看运行记录。

## 文件

- `static/index.html`: 页面结构。
- `static/styles.css`: 响应式布局和视觉样式。
- `static/app.js`: 状态管理、接口调用、渲染。

## 设计取舍

- 不引入前端框架，降低部署和维护成本。
- 页面直接调用同源 API，避免额外 CORS 配置。
- 第一屏就是工作台，不做落地页。
- 样式偏后台工具，不使用营销化大图和装饰布局。
- 品牌、连接状态、运行环境和新建入口集中在全局顶栏，出题表单使用原生 `dialog`，不持续占用题库空间。
- 桌面端题目库固定在视口内并独立滚动，运行记录保持在详情右侧。
- 中等宽度先将运行记录移到详情下方；平板将题目库压缩为双列；手机切为单列并保留紧凑首屏。
- 状态指标使用一条响应式信息带，桌面六列、窄屏三列、手机两列。
- 页面按 1440、768、390 和 320 像素宽度检查横向溢出，生成弹窗在桌面和手机视口内完整显示。
